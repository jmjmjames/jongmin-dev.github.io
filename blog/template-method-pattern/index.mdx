## 템플릿

템플릿 메서트 패턴에 앞서, 탬플릿은 기준이 되는 틀이다.

## 추상 템플릿 메서드 패턴

추상 템플릿을 통해 단독으로 사용되지 않게 사용한다. 이 때 변하지 않는 부분을 추상 템플릿 클래스에 몰아 넣고, 추상 템플릿에 내장 함수(`변하는 부분`)를 하나 만든다. 변하는 부분을 `상속`과 `Overriding`으로 해결한다.

추상메서드 타입의 구체타입의 인스턴스를 할당하고, 추상 메서드의 함수를 호출하면, 오버라이딩된 함수가 호출된다.

또한 변하지 않는 부분, 즉 공통 부분에서 변경이 일어나면 추상 템플릿에서 변경을 한 번 변경하면 된다. 이는 단일 책임 원칙을 잘 지키는 것이다.

## 익명 내부 클래스 사용하기

그러나 이는 추상 템플릿을 이용하려면, 계속 클래스를 만들어야 한다. 이를 해결하는 방법은 -> 익명 내부 클래스를 사용하면, 객체 인스턴스를 생성하면서 동시에 생성할 클래스를 상속 받은 자식 클래스를 정의할 수 있다. 이 클래스를 직접 지정하는 이름이 없고 클래스 내부에서 선언되는 익명 내부 클래스라 한다.

## 템플릿 메서드 패턴 -정의

GOF 디자인 패턴에서는 템플릿 메서드 패턴을 다음과 같이 정의했다.

> '작업에서 알고리즘의 골격을 정의하고 일부 단계를 하위 클래스로 연기합니다. 템플리 메서드를 사용하면 하위 클래스가 알고리즘의 구조를 변경하지 않고도 알고리즘의 특정 단계(변하는 부분)를 재정의할 수 있습니다.[GOF]

부모 클래스에 알고리즘 골격인 템플릿을 정의하고, 일부 변경되는 로직은 자식 클래스에 정의하는 것이다. 결국 상속과 오버라이딩을 통한 다형성으로 문제를 해결하는 것이다.
